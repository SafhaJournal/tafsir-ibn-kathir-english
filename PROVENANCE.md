# Provenance & Copyright Record — Tafsir Ibn Kathir (Safha English)

**Edition:** `ibn-kathir-en-safha` · v1
**Prepared:** 2026-07-07
**Scope:** 114 surahs · 6,236 tafsir records (English)

> This document records the source, method, and independence evidence for this English
> tafsir corpus. It exists so the provenance of the text can be established from the record
> rather than reconstructed after the fact. It is a factual/technical record, **not legal
> advice**; have qualified counsel review it before relying on it.

---

## 1. The claim, in one paragraph

The English text in this corpus is a **new, independent translation of a public-domain Arabic
work**, generated by an AI model that was shown **only the Arabic source and never any English
translation**. No copyrighted English translation was used as a source, template, or input to
the text that is distributed. A separate, cross-vendor AI model checked each surah's fidelity;
that quality check is the *only* point at which any copyrighted English edition was accessed,
it was used solely as a hidden internal reference for comparison, and **not one word of it
appears in this corpus.** Quantitative overlap measurements (Section 6) corroborate that the
distributed English is not copied from the reference.

## 2. Source text (public domain)

| | |
|---|---|
| Work | *Tafsir al-Qur'an al-'Azim* (Tafsir Ibn Kathir), Arabic, unabridged |
| Author | Isma'il ibn 'Umar ibn Kathir |
| Author died | **774 AH / 1373 CE** |
| Copyright status | **Public domain worldwide** (author died > 650 years ago) |
| Digital source | QUL resource 22, obtained via the `spa5k/tafsir_api` mirror, edition id `ar-tafsir-ibn-kathir`, from `https://raw.githubusercontent.com/spa5k/tafsir_api/main/tafsir` |

Because the underlying Arabic work has been in the public domain for centuries, anyone is free
to translate it. A faithful new translation of a public-domain work is an **independent,
separately-authored derivative**; copyright in *this* English text vests in its author/commissioner,
provided it was not copied from someone else's copyrighted translation. Sections 3–6 document that
it was not.

## 3. What is distributed — and what is not

**Distributed in this repository (`corpus/ibn-kathir-en-safha.jsonl` + `surahs/`):** only the
independently-generated English translation, plus neutral provenance fields (surah/ayah keys,
block grouping, translator model, timestamps).

**Deliberately NOT in this repository:**
- No text, sentence, or phrase copied from any copyrighted English translation.
- No copy of the copyrighted English reference edition (it lives only in the private build
  pipeline and is never redistributed).
- No fidelity scores, grader verdicts, or QA notes in the corpus rows. Per-surah fidelity is
  summarized separately in `QUALITY.md`; the detailed grade cards are retained by the maintainer,
  not published here (Section 7).

The library-build tooling writes **only our own translation and our own metrics**; the
copyrighted reference is never written into the library or the corpus by construction.

---

## 4. The independence firewall (how non-derivation is enforced, not just asserted)

The pipeline is architected so the translator **cannot** have copied a copyrighted translation,
because it never received one:

1. **Translator input = Arabic only.** The translation model (Anthropic **Claude Sonnet 5**,
   `claude-sonnet-5`; a small number of flagged revisions used Claude Opus 4.8) was given the
   public-domain Arabic block and an instruction prompt. It was **not** given, and had no means
   to retrieve, any English translation.
2. **No English ever reaches the translator's context.** The scripts that fetch source material
   log **byte counts only**, never content, so no English text is ever displayed alongside, or
   stored with, the material the translator sees.
3. **Grading is done by a different vendor's model.** Fidelity was assessed by Google **Gemini
   3.1 Pro** (`gemini-3.1-pro-preview`) — a separate model from a separate company — comparing the
   new English against the **Arabic**. Keeping translator ≠ grader preserves independence.
4. **The grader's feedback is diagnosis-only and leak-filtered.** The grader points to the Arabic
   ("this word is mistranslated relative to the Arabic") and its output is passed through an
   automated redaction filter (`redactReferenceLeaks`) that strips any reference wording before
   the feedback is stored. Grader feedback therefore cannot smuggle reference text back into the
   translation.
5. **Deterministic, non-AI gates.** Every surah also passes code-only checks: a verbatim-overlap
   "measuring machine" (Section 6), an em-dash style gate, and a foreign-script gate.

---

## 5. Use of a copyrighted reference for QA only

At exactly one step — grading — an approved published English edition of Ibn Kathir
(`en-tafisr-ibn-kathir` on the same mirror) was consulted **by the grading model, as a hidden
reference, for internal quality comparison only.** Concerning that use:

- It was **not** seen by the translator and had **no** influence on how the English was written.
- It was **not** copied into the output, the library, or this corpus.
- Only non-expressive, quantitative comparison signals and Arabic-anchored diagnoses were retained.

Using a copyrighted work internally for comparison/analysis — without reproducing its expression
in the distributed product — is a **transformative, non-expressive** use. This record documents
the factual basis for that position; a copyright attorney should confirm its application to your
jurisdiction and distribution.

## 6. Quantitative evidence of independence (the "measuring machine")

For **every** surah, the pipeline measures verbatim overlap between the distributed English and
the copyrighted reference (n-gram containment and longest common word-run). Low overlap is
affirmative evidence that the output is not a copy. Across all 114 graded surahs:

| Measure (expository prose) | Mean | Median | Max |
|---|---|---|---|
| 4-gram containment (fraction of 4-word sequences shared) | **0.088** | 0.088 | 0.164 |
| Longest common prose run | **18.4 words** | 18 | 50 |

Interpretation: on average **~91% of four-word prose sequences in our English do not appear in the
reference at all.** The residual overlap concentrates in *unavoidable shared material* — the wording
of Qur'anic verses (scripture both editions render similarly), proper names, isnad chains, and
standardized Islamic honorific formulas ("the Messenger of Allah", "may Allah be pleased with him")
— not in Ibn Kathir's expository prose, where independent phrasing shows up as the low containment
figures above. Per-surah overlap figures are recorded for every ayah in `qa/<surah>.jsonl`
(`prose_overlap_4gram`) and in `reports/<surah>.json`.

## 7. Audit trail & reproducibility

Published in this repository:

- **`surahs/<NNN>-<name>.md`** — human-reviewable translation with the **Arabic source beside each
  block**, so any reader can compare the English to the Arabic directly.
- **`QUALITY.md`** — per-surah fidelity scores and verbatim-overlap figures.
- **`PROVENANCE.md` + `provenance.json`** — this record, self-documenting and shipped with the data.

Retained by the maintainer (not published; available on request to substantiate this record):

- Per-surah grade cards — fidelity score, examiner verdict, overlap metrics, style-gate results,
  and flagged issues (each with an Arabic anchor, never reference text).
- Per-ayah QA grades and the deterministic verbatim-overlap logs.
- The fidelity-polish record (every flagged issue for surahs 2–77 verified against the Arabic;
  genuine errors corrected, grader false-positives skipped; the English reference was never
  consulted during polishing).

The copyrighted reference edition is never stored in any published *or* retained distributable —
only non-expressive comparison metrics and Arabic-anchored diagnoses are kept.

**Chain of custody (summary):**

| Stage | Actor | Input | Output |
|---|---|---|---|
| Fetch | scripts | public-domain Arabic (QUL 22) | Arabic blocks; byte-count logs only |
| Translate | Claude Sonnet 5 (Anthropic) | Arabic only | English (this corpus) |
| Grade | Gemini 3.1 Pro (Google) | Arabic + English + hidden reference | Arabic-anchored diagnoses, overlap metrics (leak-filtered) |
| Gate | deterministic code | English + reference | overlap %, style pass/fail |
| Publish | build/export scripts | our English + our metrics | `surahs/`, `corpus/*.jsonl` (no reference text) |

## 8. Summary statement

> The English translation of Tafsir Ibn Kathir in this corpus was generated independently from the
> public-domain Arabic source by an AI translator that never received any copyrighted English
> translation. A copyrighted English edition was used only as a hidden reference by a separate
> grading model for internal quality comparison, was never reproduced in the output, and does not
> appear anywhere in this corpus. Measured verbatim overlap with that reference is low and confined
> to unavoidable shared scriptural and formulaic material. The distributed English is an original
> derivative of a public-domain work.
